# ComplianceAI

HIPAA compliance automation for healthcare practices. Solo practitioners and small clinics get a continuously updated compliance dashboard, automated practice-specific policy documents, incident response workflows, and audit-ready reports.

## What It Does

- **Live Compliance Score** — Real-time score across all 4 HIPAA safeguard domains (administrative, physical, technical, organizational)
- **AI Policy Generator** — Practice-specific policies generated by Claude claude-sonnet-4-6, not generic templates
- **Incident Response System** — Log any breach, get AI response workflows + automatic breach notification assessment
- **Audit-Ready Reports** — One-click compliance reports for HHS investigations
- **Dashboard** — Interactive HIPAA compliance checklist with per-category status tracking

## Setup

### 1. Install dependencies

```bash
npm install
```

### 2. Configure environment variables

Copy `.env.example` to `.env.local` and fill in your keys:

```bash
cp .env.example .env.local
```

Required variables:
- `NEXT_PUBLIC_SUPABASE_URL` — Your Supabase project URL
- `NEXT_PUBLIC_SUPABASE_ANON_KEY` — Your Supabase anon key
- `ANTHROPIC_API_KEY` — Your Anthropic API key
- `STRIPE_SECRET_KEY` — Stripe secret key (for payments)
- `NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY` — Stripe publishable key

### 3. Set up Supabase

Run the schema in your Supabase SQL editor:

```bash
# Copy contents of supabase/schema.sql and run in Supabase dashboard
```

### 4. Run development server

```bash
npm run dev
```

Open http://localhost:3000 in your browser.

## API Routes

### POST /api/generate-policy

Generate a HIPAA policy document using Claude.

Body:
```json
{
  "documentType": "privacy_policy",
  "practiceInfo": {
    "name": "Practice Name",
    "type": "medical",
    "state": "CA",
    "providerCount": 2
  }
}
```

Document types: `privacy_policy`, `security_policy`, `workforce_training`, `baa_template`, `risk_assessment`, `sanctions_policy`

### POST /api/compliance-score

Calculate weighted compliance score from compliance area statuses.

Body:
```json
{
  "complianceAreas": [
    { "area": "administrative", "category": "Risk Analysis", "status": "compliant" }
  ]
}
```

### POST /api/incident-response

Generate HIPAA incident response guidance for a reported incident.

Body:
```json
{
  "incidentType": "Unauthorized access to PHI",
  "description": "...",
  "affectedCount": 45,
  "discoveryDate": "2026-03-21",
  "practiceInfo": { "name": "...", "type": "medical", "state": "CA" }
}
```
