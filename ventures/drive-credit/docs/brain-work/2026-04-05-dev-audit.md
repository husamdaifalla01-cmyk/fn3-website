## 3. /api/track-click (src/app/api/track-click/route.ts)

**Status: BUG — filesystem writes fail on Vercel**

The route appends click data to `click-log.jsonl` using `fs.appendFile`. On Vercel's serverless/edge runtime, the filesystem is read-only and ephemeral. This means:
- Writes may silently fail or throw errors.
- Even if they succeed, data is lost on the next cold start.
- The `catch` block returns a 500, so the client sees errors intermittently.

**Impact:** Low for users (sendClickToApi is fire-and-forget, never blocks UX), but we lose all click attribution data. This defeats the purpose of MaxBounty subid tracking.

**Recommended fix:** Replace filesystem logging with one of:
1. Vercel KV / Upstash Redis (simplest, free tier available)
2. Supabase insert (already have Resend, adding another SaaS is fine)
3. POST to a Google Sheets webhook (zero-cost, good enough for MVP volume)
4. Log to Vercel's stdout and use Log Drain to forward to a store
