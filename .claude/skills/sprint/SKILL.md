# /sprint - Sprint Management Skill

## Description
Plan, execute, and review development sprints. Coordinates all instances toward shared sprint goals with championship-level execution.

## Commands

### `/sprint plan [goal]`
Create a new sprint focused on a goal with success criteria, instance assignments, dependencies, and checkpoints.

### `/sprint start`
Begin the current sprint and activate all instances.

### `/sprint status`
Check sprint progress including task status, blockers, velocity, and next actions.

### `/sprint review`
End-of-sprint review with delivered items, metrics, and lessons learned.

### `/sprint retro`
Deep retrospective: Stop/Start/Continue analysis and action items.

---

## Sprint Rules

1. **One Goal**: Each sprint has ONE primary goal
2. **Clear Success**: Know exactly what "done" looks like
3. **All Hands**: Every instance contributes
4. **No Scope Creep**: New ideas go to backlog
5. **Ship It**: Incomplete > Unstarted
